## Make
### Definitions
- **Target** - a node in the dependency graph. Usually a file generated by commands or a label for commands to execute (phony target).
- **Dependencies** - files or other targets that the current target depends on. Before updating a target, make recursively processes its dependencies
and then decides whether the target should be rebuilt by comparing timestamps: 
```
for dependency D:
    if D has build rule:
        recursively update(D)
    else:
        treat D as leaf filesystem file

THEN after all dependencies processed:
compare timestamps for current target
```
- **Commands/Recipes** - shell commands to execute by make. If on the next line after target/dependencies, should be started with **tab**.
- **Rule** - individual target + dependencies + commands  

Rule example:
```makefile
# Rule 1
target: dependencies
  commands
  
# Rule 2, different syntax
target: dependencies ; commands
```
### Variables
- Variables are strings (`var := a b` means string "a b", `var := 'a b'` means string "'a b'")
- Undefined variable is an empty string 
- Define variable `var := value`
- Define variable, but the value is set only when it's used `var = value` ([example](https://makefiletutorial.com/#flavors-and-modification))
- Append to variable `var += value`
- Reference a variable `${varname}` or `$(varname)`
- Command-line arguments in `make VARNAME=value` are also variables, in Makefile you can access it with `${VARNAME}`
- To make a variable visible for child processes use `export` keyword.
### Other
- Make is language-independent. It executes arbitrary commands and resolves dependencies between targets.
- Running `make` without arguments will choose the first target. 
- `clean` and `all` targets are standard names, not keywords. 
- `.PHONY` - special target that marks other targets as not representing real files. Phony targets are always executed when requested.
- Commands echo by default, to suppress it use `.SILENT`:
```makefile
.SILENT: # no echoing for a whole makefile
.SILENT: target # no echoing for a specific target
target:
  @command # add @ to silent specific command
```
- If target's commands are on different lines, they are executed in different shells.
